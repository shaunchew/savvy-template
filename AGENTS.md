# savvy-framework

A self-policing AI-dev framework shipped as a Claude Code plugin (`sf`). This repo IS the plugin: the root payload is the authored source, published as marketplace `shaunchew/savvy-template`.

## Stack

- Language: Bash (3.2-compatible — macOS system bash) + jq
- Framework: Claude Code plugin architecture (commands/, skills/, hooks/, agents/, .claude-plugin/)
- Runtime: macOS `/bin/bash` and Linux bash (CI runs both)
- Datastore: none — plain files, git as the ledger
- Deploy target: GitHub Releases (tag-triggered `release.yml`) + Claude Code plugin marketplace

## Commands

```
# Setup
(none — zero dependencies beyond bash 3.2+, git, jq)

# Build (after ANY engine edit)
bash scripts/build-plugin.sh   # regenerates template/ + skeleton/ + manifest

# Test
bash tests/run.sh              # must be green before any commit to main

# Release
# bump VERSION + CHANGELOG, run build-plugin.sh, tag vX.Y.Z, push tag
# release.yml gates: tag↔VERSION↔plugin.json↔CHANGELOG, suite, drift
```

## Conventions

- Repo root payload is the ONLY authored source. `template/` and `skeleton/` are GENERATED by `scripts/build-plugin.sh` — never hand-edit them.
- `build-plugin.sh` must run after any engine edit and its regenerated artifacts committed; CI and the release gate both enforce drift.
- Every change to `scripts/`, `hooks/`, or `migrations/` needs a regression test in `tests/`.
- bash 3.2 compatibility is mandatory (no associative arrays, no `${var,,}`, etc.).
- Test fixtures that look like secrets must be RUNTIME-ASSEMBLED, never literal (GitHub push protection blocks pushes).
- New scripts need `chmod +x` (files are created 0644 by default; tests fail with exit 126 otherwise).

## Negative rules

- NEVER delete user files in adopt/eject/upgrade paths — quarantine to `.claude/.savvy-detached-<ts>/`.
- NEVER let hooks act outside adopted projects (gate on `[framework]` in `.claude/config.toml`). The secret-scan floor guard is the single deliberate everywhere-exception.
- NEVER hand-edit `template/` or `skeleton/`.
- NEVER touch the real `~/.claude` from tests — isolate with `HOME=/tmp/... CLAUDE_CONFIG_DIR=...`.
- NEVER commit secret-shaped literals, even obviously fake ones.
